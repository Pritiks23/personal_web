        [DDP]

Distributed Training Pipeline — PyTorch DDP

      

        A distributed deep learning training pipeline across 2× NVIDIA GeForce RTX 3090 GPUs,
        built on PyTorch Distributed Data Parallel with NCCL as the communication backend.
        Broadcast-based weight synchronization and rank-based process isolation keep every
        worker's parameters identical after every step.
      

      
    
        
  • Engineered a DistributedSampler pipeline that partitions all 60,000 CIFAR-10 images into non-overlapping mini-batches per GPU, eliminating redundant work.
    • 
        
  • Synchronized gradient aggregation via NCCL all-reduce, verified for 100% parameter consistency across workers.
    • 
        
  • Scaled data loading and model sync with torch.multiprocessing across ResNet-based training runs.
    •

Tensor Parallelism Across Multiple GPUs

      

        A 2-GPU tensor-parallel architecture addressing transformer inference scalability limits,
        built on PyTorch and NCCL. Distributed GPT-style blocks — LayerNorm, TP attention,
        TP MLP — shard computation across devices while keeping outputs numerically stable.
      

      
    
        
  • Validated stable [B, T, 768] outputs across 30 inference iterations under sharded computation.
    • 
        
  • Built an end-to-end distributed text embedding pipeline: tokenization → tensor-parallel backbone → pooling → projection.
    • 
        
  • Generated 256-dimensional embeddings for real-world text workloads split across multiple GPUs.

        [K8s]
        ● 7 incidents resolved
      

      
GPU Platform Failure Investigation Lab

      

        A Kubernetes-based distributed training failure lab that reproduces real GPU platform
        incidents — without needing physical GPUs. Built with PyTorch Jobs, indexed multi-rank
        workloads, headless Services, and GPU extended resources (nvidia.com/gpu),
        so incident response can be trained end-to-end.
      

      
    
        
  • Reproduces CUDA OOM, NCCL collective timeouts, CrashLoopBackOff, ImagePullBackOff, GPU scheduling failures, PVC mount errors, and training deadlocks.
    • 
        
  • Built an observability stack — Prometheus, Grafana, DCGM-style GPU telemetry, Kubernetes Events, pod lifecycle monitoring, alert rules, automated triage.
    • 
        
  • Authored incident-response playbooks and postmortems using kubectl, scheduler events, pod exit codes, and PVC binding/image-pull diagnostics.
